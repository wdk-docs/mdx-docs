## 显示案例

数千个有趣的项目都是由 unified、混合和匹配的积木组成的。这里展示了几个用 unified 制作的有趣案例。

## 团队

unified 集体是一个由组织组成的联邦系统，依次由项目组成，由指导项目的团队成员管理。成员通过寻求共识的决策模式管理集体，详见 [unifiedjs/collective](https://github.com/unifiedjs/collective/)。

## 赞助商

维护集体、开发新项目、保持一切快速安全以及帮助用户是一项艰巨的工作。财政支持让团队花更多时间维护现有项目和开发新项目。要支持 unified，请成为 [OpenCollective](http://opencollective.com/unified) 的赞助商或支持者。

## 支持

在使用 unified 时，有几个地方可以获得支持，如本节所述。在加入 unified 的社区之前，请阅读我们的[行为准则](https://github.com/unifiedjs/.github/blob/HEAD/code-of-conduct.md)。我们希望所有社区成员都遵守内部的指导方针。

可以在每个组织的 GitHub 讨论中提问。查看 [support.md](https://github.com/unifiedjs/.github/blob/HEAD/support.md)，了解如何帮助我们帮助您。花足够的时间让你的问题清晰而详细是很重要的，因为一般来说，如果花更多的时间在一个高质量的问题上，那么给出一个好答案所需的时间就会更少。

问题可以在我们组织下的一个存储库中的 GitHub 上提出。请参阅 [contributing.md](https://github.com/unifiedjs/.github/blob/HEAD/contributing.md) 了解如何报告问题。本文档还解释了如何改进文档并为 unified 做出贡献。

安全问题可以通过电子邮件报告，并通过 GitHub 上的安全咨询进行处理。请参阅 security.md 了解如何提交报告。

## 学习

要了解有关 unified 的信息，请浏览我们的[`了解`](https://unifiedjs.com/learn/)部分。本节包括几篇文章，从完成小型特定任务的食谱到如何完成大型任务的指南。此外，我们项目的自述文件（可通过 [Explore](https://unifiedjs.com/explore/) 部分或 GitHub 和 npm 获得）详细描述了每个项目。

## 消息

关注推特上的 [@unifiedjs](https://twitter.com/unifiedjs) 账号获取新闻。你也可以在这个账号上发推特，提出问题或建议，或者在你用 unified 做某事时提及它！

有关最新版本，请参阅[`版本`](https://unifiedjs.com/explore/release/)部分。
