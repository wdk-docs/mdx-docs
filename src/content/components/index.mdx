MDX 的一个优点是，您可以写下标记并指定要使用的组件，而不是 HTML 元素。
下表列出了那些可以被覆盖的 HTML 元素、所需的标记以及普通的 JSX 等价物。
但是，以`a`的第一行为例，您可以通过以下操作将该超链接替换为`<FancyLink>`：

```tsx
import Readme from "./readme.md"; // Assumes an integration is used to compile MDX -> JS.
import { FancyLink } from "./components/fancy-link.js";

<Readme components={{ a: FancyLink }} />;
```

有关如何使用降价的更多信息，请参阅[CommonMark][].

可以覆盖的组件使用它们的标准 HTML 名称。
通常，在 markdown 中，这些名字是: `a`, `blockquote`, `br`, `code`, `em`,
`h1`, `h2`, `h3`, `h4`, `h5`, `h6`, `hr`, `img`, `li`, `ol`, `p`, `pre`,
`strong`, 和 `ul`.
[`remark-gfm`][gfm] ([请参阅指南][guide-gfm]), 你也可以使用: `del`,
`input`, `section`, `sup`, `table`, `tbody`, `td`, `th`, `thead`, 和 `tr`.
其他注释插件添加了对新构造的支持，并宣传它们可以使用 rehype，也可以使用 MDX。

有关组件的更多信息，请参阅[¶使用 MDX 中的组件][components].
